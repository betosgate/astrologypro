# Module 03 - Make Trigger Completion the Authoritative Lesson Gate

- Status: Completed (2026-04-08, verified)
- Completion Notes: Trigger answer route src/app/api/trainee/training/lessons/[id]/triggers/[triggerId]/answer/route.ts writes lesson_trigger_progress; rewatch route handles failure path; lesson detail route exposes trigger completion state.

## Objective
Move lesson completion authority from the legacy full-lesson quiz endpoint to the trigger-based lesson progression model.

## Why This Task Exists
Trigger progress exists, but final lesson completion still depends on older quiz behavior. That makes the new learner flow incomplete and allows bypasses for trigger-enabled lessons.

## Current Repo State
- Trigger answer flow updates `lesson_trigger_progress`.
- Legacy `/api/trainee/training/lessons/[id]/quiz` still marks lessons complete directly on pass.
- Lesson/category/program completion and graduation logic still rely on `lesson_completions` generated by the old path.
- The trainee lesson page still receives both legacy quiz data and trigger data from `/api/trainee/training/lessons/[id]`, so both flows currently coexist in the live lesson payload.

## Exact Gap
- Trigger progression is not the authoritative completion source.
- A learner can still complete a lesson through the legacy quiz path even if trigger behavior exists.

## Fixed Behavior Decisions
- For lessons with active trigger records, authoritative completion comes from trigger progress only.
- For lessons without active trigger records, the legacy full-lesson quiz flow may remain as fallback.
- `lesson_completions` remains the canonical completion record, but trigger-enabled lessons must reach it through trigger completion logic.
- Repeated trigger submissions and repeated completion checks must remain idempotent.
- The lesson-detail payload may continue exposing legacy quiz data for backward compatibility, but trigger-enabled lessons must no longer be completable through that legacy quiz path.

## Required Implementation
- Define this completion rule:
  - if a lesson has active trigger records, the lesson completes only when all required triggers are passed
  - if a lesson has no active trigger records, legacy quiz behavior may remain as fallback
- Update trigger answer flow so it can detect when all required triggers are passed and then:
  - complete the lesson
  - update category completion if applicable
  - update program progress caches as needed
- Update legacy full-lesson quiz route so it does not mark a trigger-based lesson complete.
- Ensure idempotency remains intact for repeated trigger submissions or repeated completion checks.
