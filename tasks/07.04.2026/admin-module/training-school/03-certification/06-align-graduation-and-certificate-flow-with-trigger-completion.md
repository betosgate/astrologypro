# Module 06 - Align Graduation and Certificate Flow With Trigger Completion

## Objective
Ensure graduation and certificate issuance follow the final trigger-based lesson completion model rather than the older legacy quiz-completion path.

## Why This Task Exists
Once trigger completion becomes authoritative for trigger-enabled lessons, graduation and certificate logic must follow that same source of truth. Otherwise users can graduate through the wrong path or receive certificates too early.

## Current Repo State
- Graduation, `graduated_at`, and `certificate_code` already exist.
- Lesson completion and certificate notification logic exists.
- The current certificate flow still depends on lesson completion generated by the legacy completion path in places.
- Graduation is currently checked in two different ways:
  - `/api/trainee/training/lessons/[id]/complete` auto-graduates from `user_program_progress`
  - `/api/trainee/check-graduation` counts all active lessons against `lesson_completions`

## Exact Gap
- Even if trigger playback is completed, graduation/certificate behavior is not yet fully aligned with that new authoritative path.
- This risks granting certificates based on the wrong completion source.

## Fixed Behavior Decisions
- Graduation and certificate issuance must depend on authoritative lesson completion only.
- For trigger-enabled lessons, authoritative lesson completion comes from trigger completion logic introduced in Module 03.
- Existing certificate verification fields and routes remain in place.
- Certificate issuance and graduation checks must remain idempotent.
- The implementation must converge the existing graduation paths onto one consistent completion rule instead of leaving both paths to diverge.

## Required Implementation
- Revalidate all graduation triggers so they depend on authoritative lesson completion only.
- Once Module 03 is complete, ensure certificate issuance is driven by trigger-based completion for trigger-enabled lessons.
- Keep current certificate verification route and existing certificate fields intact.
- Preserve idempotency so repeated completion checks do not re-issue graduation/certificate state.

## Acceptance Criteria
- Trigger-enabled lessons contribute to graduation only through the new authoritative completion path.
- Certificates are not issued prematurely through the old full-lesson quiz path.
- Verification and certificate access still work after graduation.
- Repeated graduation checks do not create duplicate certificate state or duplicate issuance side effects.
- `/api/trainee/training/lessons/[id]/complete` and `/api/trainee/check-graduation` no longer produce contradictory graduation outcomes for the same user state.
