Mundane Astrology Dashboard — Requirement Analysis Document 1) Executive Summary This product is a professional mundane astrology intelligence platform for astrologers who analyze countries, leaders, global events, markets, weather pattern( skyscript.co.uk )nt correlations . It should not be treated like a normal natal astrology app with a few extra charts. A full-fledged mundane platform must work like a research, forecasting, charting, timeline, geo-analysis, and publication system . For a Next.js full-stack implementation, the system should be designed as a modular astrology workbench with: - highly accurate chart calculation - multi-entity chart management - geo and time-aware analysis - event and forecast tracking - research database and backtesting - collaboration and publishing workflows - strong auditability and reproducibility 2) Product Vision Enable a professional mundane astrologer to: - study nations, cities, leaders, institutions, commodities, markets, and world events - compare multiple charts at once - detect high-risk or high-importance periods - create structured forecasts with evidence - maintain a research archive of past events and predictions - publish dashboards, reports, alerts, and client-ready outputs 3) Core Product Assumption Because astrologers work differently, the platform must support a framework-driven architecture : - Western mundane astrology mode - Vedic / Mediniya astrology mode - Hybrid mode for astrologers who combine both This should be configurable at: - global system level - astrologer profile level - workspace or project level - chart or forecast level That means the platform must never hardcode only one tradition. 4) Primary Users 4.1 Professional Mundane Astrologer Uses the system daily to analyze countries, leaders, political developments, eclipses, ingresses, markets, weather patterns, and historical cycles. 4.2 Research Astrologer / Analyst Supports data entry, event tagging, archive building, chart comparisons, and backtesting. 4.3 Editorial / Publication User Prepares forecasts, newsletters, client reports, dashboards, and public/private output pages. 4.4 Admin / System Manager Manages users, permissions, datasets, calculations, API keys, content moderation, and subscription plans. 4.5 Optional Client / Subscriber Viewer Reads curated outputs, dashboards, alerts, and forecast summaries with limited access. 5) Business Goals The product should allow the astrologer or astrology business to: - sell premium research subscriptions - sell country-specific or market-specific forecast feeds - publish internal or public reports - maintain a proprietary historical database - deliver client dashboards and alerts - differentiate through deeper mundane research, not just generic horoscope content 6) What Makes This Product Gold Standard A weak product gives charts. A strong product gives interpretation helpers. A gold-standard product gives context, comparison, evidence, tracking, and recall . So the system must include: - multi-chart comparison - time-range forecasting - world map overlays - country and leader profile repositories - historical event timeline correlation - forecast journal with evidence - alerting and watchlists - backtesting against actual events - note-taking and publication workflow - calculation reproducibility and chart versioning 7) High-Level Modules Authentication and account system User roles and permissions Dashboard home Mundane chart studio Country / entity registry Leader chart registry Event calendar and astronomical timeline Forecast engine and signal scoring Geo and world map intelligence Historical event database Research workspace and notes Alerts and watchlists Publication and report builder Data import and external feeds Backtesting and accuracy tracker Admin and system controls Billing / subscriptions (optional if SaaS) 8) Detailed Functional Requirements 8.1 Authentication and Identity Features email/password login social login if desired optional OTP / MFA session management device/session activity list password reset invite-based onboarding for teams Roles super admin admin astrologer researcher editor client viewer Requirements RBAC at page, action, and data level per-workspace access control audit logging for critical actions 8.2 Dashboard Home This is the operating center.
