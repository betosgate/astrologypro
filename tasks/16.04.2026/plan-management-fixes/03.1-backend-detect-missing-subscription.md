# 03.1 Backend: Detect Missing Subscription - 2026-04-16

## Goal

Modify the `change-tier` API to identify users who do not have an active Stripe subscription and prevent "free" upgrades by signaling a checkout requirement.

## Context

The current `src/app/api/community/plan/change-tier/route.ts` logic updates the database even if the user has no `stripe_subscription_id`. This allows users to bypass payment.

## Files To Change

| File | Change |
|---|---|
| `src/app/api/community/plan/change-tier/route.ts` | Add a check for `stripe_subscription_id` presence. |

## Required Behavior

1. After fetching the `member` row, check if `member.stripe_subscription_id` exists.
2. If it is null or empty:
    - Do **NOT** call `.update()` on `community_members`.
    - Do **NOT** perform any Stripe item updates.
    - Return a 200 OK response with:
      ```json
      { 
        "requires_checkout": true, 
        "plan_type": "family" | "individual" 
      }
      ```
3. If it exists, proceed with the existing upgrade/downgrade logic.

## Acceptance Criteria

- [ ] API returns `requires_checkout: true` for users without a subscription.
- [ ] Database `pm_tier_id` is NOT modified for users without a subscription.
