## Goal

Enable the frontend to redirect the user to a Stripe Checkout session if the `change-tier` API indicates that a new subscription is required.

## Context

Previously, the frontend assumed `change-tier` always succeeded without a separate checkout flow. For new or resubscribing members, we must catch the `requires_checkout` flag and trigger the actual payment session.

## Files To Change

| File | Change |
|---|---|
| `src/app/community/plan/page.tsx` | Update `handleTierSwitch` to handle redirection. |

## Required Behavior

1. In the `handleTierSwitch` function:
    - After `const d = await res.json()`, check if `d.requires_checkout` is truthy.
2. If `requires_checkout`:
    - Display a "Redirecting to checkout..." toast.
    - Make a second POST request to `/api/community/checkout`.
    - Body: `{ membershipType: "perennial_mandalism", planType: d.plan_type }`.
    - Extract `url` from the checkout response.
    - Redirect: `window.location.href = url`.
3. If NOT `requires_checkout`, proceed with the existing "Success" toast and `loadPlan()` refresh.

## Acceptance Criteria

- [ ] Users without a subscription see a Stripe payment page after clicking "Switch to This Plan".
- [ ] Users with a subscription still get an instant update (no redirect).
