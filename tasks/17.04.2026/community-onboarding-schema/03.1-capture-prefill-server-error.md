## Goal

Capture the exact server-side error for `GET /api/community/onboarding/prefill` before changing the route or database schema.

## Endpoint

```txt
GET /api/community/onboarding/prefill
```

Page route:

```txt
/community/onboarding
```

Observed status:

```txt
500 Internal Server Error
```

## Why This Matters

The client only sees a generic failure. The route currently logs specific errors with these prefixes:

```txt
[onboarding/prefill] lookup error:
[onboarding/prefill] family lookup error:
[onboarding/prefill] error:
```

The exact log determines whether the failure is:

- a missing `community_members` column, such as `birth_country` or `occupation`
- a missing `community_family_members` column, such as `invite_email` or `user_id`
- an auth/session issue
- an unexpected runtime exception

## Steps

1. Start the app server in the target environment.
2. Sign in as a community member who reproduces the issue.
3. Open `/community/onboarding`.
4. Watch server logs while the page requests `/api/community/onboarding/prefill`.
5. Record the exact error prefix, Supabase error message, code, details, and hint.
