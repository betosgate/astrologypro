## Goal

Update `GET /api/community/onboarding/prefill` so the endpoint has a clear, stable response contract and does not return `500` for expected empty onboarding states.

## File To Change

```txt
src/app/api/community/onboarding/prefill/route.ts
```

## Required Response Contract

Unauthenticated:

```json
{
  "error": "Unauthorized"
}
```

Status:

```txt
401
```

Authenticated, no member row:

```json
{
  "member": null,
  "family_members": []
}
```

Status:

```txt
200
```

Authenticated, member row exists:

```json
{
  "member": {
    "id": "...",
    "birth_country": "..."
  },
  "family_members": []
}
```

Status:

```txt
200
```

Unexpected database or runtime failure:

```json
{
  "error": "Failed to load profile."
}
```

Status:

```txt
500
```

## Implementation Notes

- Keep the route wrapped in `try/catch`.
- Log Supabase errors with enough detail to debug the exact table/column problem.
- Return empty data only for expected missing records, not for actual query errors.
- If the exact server error is a missing column, fix schema or schema cache first.
- Coordinate with Task 2 if `birth_country` is the root cause.

## Acceptance Criteria

- [ ] Missing member row returns `200` with `member: null`.
- [ ] Empty household records return `family_members: []`.
- [ ] Real database errors still return controlled `500` with server logs.
- [ ] The response shape matches what `/community/onboarding` expects.
- [ ] No schema dependency is hidden by swallowing query errors.
