## Goal

Verify that `POST /api/community/onboarding/complete` succeeds with a valid authenticated payload containing `"birth_country":"Capitol Planning Region"`.

## Preconditions

- Tester is authenticated as a real user.
- The user has a `community_members` row with:
  - `membership_type = 'perennial_mandalism'`
  - `membership_status = 'active'`
- `community_members.birth_country` exists in the active database.
- Supabase/PostgREST schema cache has been refreshed if needed.

## Test Payload

The endpoint validates many required fields before the database update. Use a complete payload:

```json
{
  "first_name": "Test",
  "last_name": "Member",
  "email": "test.member@example.com",
  "phone": "5551234567",
  "gender": "other",
  "state": "CA",
  "city": "Sacramento",
  "zip": "95814",
  "address": "100 Test Street",
  "occupation": "QA Tester",
  "date_of_birth": "1990-01-01",
  "birth_time": "12:00",
  "birth_city": "Sacramento",
  "birth_country": "Capitol Planning Region"
}
```

## Browser Test Steps

1. Sign in as the test community member.
2. Open `/community/onboarding`.
3. Fill the onboarding form with the test value:

   ```txt
   Birth Country: Capitol Planning Region
   ```

4. Submit the form.
5. Confirm the request to `/api/community/onboarding/complete` returns `200`.
6. Confirm the response body is:

   ```json
   { "success": true }
   ```

## API Test Notes

If testing with `curl` or a script, include the authenticated session cookies. A request without cookies will return `401 Unauthorized` and does not validate this task.

## Acceptance Criteria

- [ ] The endpoint returns `200`.
- [ ] The response includes `{ "success": true }`.
- [ ] The old error about missing `birth_country` in the schema cache does not appear.
- [ ] Validation errors are not caused by missing required fields in the test payload.
