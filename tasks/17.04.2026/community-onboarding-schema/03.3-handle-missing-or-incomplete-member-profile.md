## Goal

Define and implement safe behavior when the authenticated user has no complete `community_members` profile yet.

## Current Behavior

In `src/app/api/community/onboarding/prefill/route.ts`, the route currently returns:

```json
{ "error": "No community membership found." }
```

with status:

```txt
404
```

For expected onboarding states, this can break the prefill flow. The page can safely render an empty form when no prefill data exists.

## Required Contract

For an authenticated user with no prefillable member record, return:

```json
{
  "member": null,
  "family_members": []
}
```

with status:

```txt
200 OK
```

Keep `401 Unauthorized` for unauthenticated requests.

## Steps

1. Inspect how community memberships are normally provisioned.
2. Confirm whether missing `community_members` during onboarding is an expected state for invited/admin-created users.
3. Update the prefill route so `!member` returns `200` with a safe empty response.
4. Do not hide real database query errors. If Supabase returns an error, log it and return a controlled error.

## Acceptance Criteria

- [ ] Authenticated user with no member record receives `200`.
- [ ] Response body is `{ "member": null, "family_members": [] }`.
- [ ] Unauthenticated user still receives `401`.
- [ ] Real query errors are still logged with enough detail to debug.
