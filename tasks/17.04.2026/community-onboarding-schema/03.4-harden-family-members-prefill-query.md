## Goal

Ensure household member prefill does not crash the onboarding page when there are no household records or when optional invite columns are unavailable in the target database.

## Current Behavior

The route queries `community_family_members` after a member is found:

```txt
id, full_name, date_of_birth, birth_time, birth_city, birth_country,
relationship, notes, invite_email, user_id
```

If any selected column is missing from the active database or schema cache, Supabase/PostgREST can return an error and the endpoint returns `500`.

## Required Behavior

- Empty household records should return:

  ```json
  "family_members": []
  ```

- Missing optional data should not crash the whole prefill flow.
- Missing required schema should be fixed by migration/cache refresh, not silently ignored.

## Steps

1. Confirm `community_family_members` exists in the active database.
2. Confirm `birth_country`, `invite_email`, and `user_id` exist and are visible to schema cache.
3. If optional invite columns are not guaranteed in all environments, decide whether the prefill endpoint needs a narrower select or a migration dependency.
4. Keep the returned `family_members` shape stable for `src/app/community/onboarding/page.tsx`.

## Acceptance Criteria

- [ ] Users with no household members receive `family_members: []`.
- [ ] Users with household members receive existing rows in creation order.
- [ ] Missing optional columns are either migrated or removed from the select only with product/backend approval.
- [ ] Family lookup errors are logged separately from primary member lookup errors.
