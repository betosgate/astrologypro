# 05 All Services Landing Coverage and Admin Governance

## Goal

Ensure every service can have landing-page coverage and that admin can govern which landing types are active.

## Product Rule

All services need landing pages.

That means the platform needs a governed model for:

- which services get service-only pages
- which services get diviner-specific landing pages
- whether both models coexist for the same service

## Recommended Admin Controls

- enable or disable service-only landing
- enable or disable diviner-specific landing
- manage service copy and metadata
- control featured diviners where applicable

## Deliverables

- landing coverage policy for all services
- admin governance requirements
- service publication rules
