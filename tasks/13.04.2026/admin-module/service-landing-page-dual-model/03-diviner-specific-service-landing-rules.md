# 03 Diviner Specific Service Landing Rules

## Goal

Preserve and formalize the direct-conversion model for service pages already tied to a specific diviner.

## Current Repo Grounding

The existing route shape already supports this model through:

- diviner username
- service slug
- service-scoped booking

## Required Rule

When a landing page is already tied to a specific diviner:

- do not ask the user to choose a diviner
- emphasize the selected diviner's credibility, style, and availability
- keep the booking path direct

## Deliverables

- no-selector rule for diviner-tied pages
- direct booking flow requirements
- diviner emphasis content rules
