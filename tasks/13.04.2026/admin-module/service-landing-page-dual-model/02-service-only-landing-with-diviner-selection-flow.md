# 02 Service Only Landing With Diviner Selection Flow

## Goal

Define how a service-only landing page lets the customer choose a diviner.

## Required Behavior

On the service-only page, the customer should be able to choose a diviner by:

- availability
- profile

## Recommended Selection Modes

### Choose by availability

Show diviners offering the service with the soonest available times.

### Choose by profile

Show diviners offering the service with profile cards, expertise, and CTA into profile or booking.

## Recommended Product Flow

1. user lands on service-only page
2. user understands the service offer
3. user chooses a diviner
4. user continues into service-scoped booking with the selected diviner

## Deliverables

- diviner-selection flow
- availability-first and profile-first variants
- handoff rules into booking
