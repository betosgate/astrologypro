# 04 Diviner Block Content and Selection UX

## Goal

Define the diviner-specific block that should appear on service landing pages.

## Required Product Need

The user asked for a diviner-specific block on these pages.

That block should help answer:

- who is offering this service
- why choose this diviner
- how to compare or proceed

## Recommended Block Contents

- diviner name and avatar
- rating or trust indicators if available
- specialty alignment to the service
- next availability snapshot
- CTA to profile
- CTA to book

## Service-Only Variant

On service-only pages, the diviner block becomes a repeatable selector card.

## Diviner-Tied Variant

On service-plus-diviner pages, the diviner block becomes a credibility and conversion module.

## Deliverables

- diviner block component rules
- card content standards
- selection and conversion UX behavior
