# 02 Dashboard Block Priority Copy and CTA

## Goal

Define the content and action structure of the emphasized dashboard block so it is clear, urgent, and useful.

## Product Requirement

The dashboard block must feel important enough that users act on it.

That means:

- strong headline
- concise explanation
- explicit missing items
- one clear CTA

## Recommended Block Contents

- status headline
  - example: `Complete your profile to unlock your charts`
- short supporting text
- bullet or token list of missing fields
- primary CTA
  - `Complete Profile`
  - `Finish Birth Details`
  - `Finish Onboarding`

## CTA Routing Rule

The CTA must go to the exact corrective flow, not a generic destination.

Examples:

- missing birth data → `/community/profile`
- onboarding incomplete → `/community/onboarding`
- missing invited-user acceptance → relevant family or household page

## Priority Placement

The block should appear above routine dashboard content when the user is blocked.

It should not sit low in the page where it competes with normal content cards.

## Deliverables

- copy hierarchy
- CTA matrix by missing-data type
- priority placement requirements
