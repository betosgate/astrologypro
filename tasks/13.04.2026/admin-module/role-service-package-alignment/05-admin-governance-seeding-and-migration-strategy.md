# 05 Admin Governance Seeding and Migration Strategy

## Goal

Define how admin manages the package model and how existing users are mapped safely.

## Admin Requirement

Admin must be able to:

- manage the `3` package types
- assign or change package type for diviners and trainees
- seed defaults
- migrate existing users without category drift

## Migration Concern

Existing users may already have:

- astrology services
- tarot services
- both

The migration strategy must define whether they map by:

- existing service inventory
- purchased plan
- admin review

## Deliverables

- admin control-surface requirements
- seed strategy
- existing-user migration rules
