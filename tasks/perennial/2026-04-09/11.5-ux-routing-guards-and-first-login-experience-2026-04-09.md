## Goal

Guide Single-plan users smoothly from signup to profile completion without confusion.

## Required UX Flow

1. Website landing -> `GET START` -> Perennial choose-plan screen at `/perennial-signup`.
2. User sees `Single`, `Couple`, and `Family` plan cards first.
3. User selects `Single`, then the page scrolls to or reveals the signup form below.
4. Single signup success -> login.
5. On first login, if completion missing, redirect to completion form.
6. After completion, route to normal Perennial dashboard.

## Guard Rules

1. If Single and missing required completion fields: force completion route.
2. If completion done: allow dashboard and chart actions.
3. Do not trap user in redirect loop.

## UX Messages

1. Explain why additional profile is required.
2. Show progress indicator: `Basic Signup Done` -> `Profile Completion` -> `Natal Ready`.
3. Keep the existing post-login experience unchanged after completion.

## Acceptance

1. New Single user is redirected to completion until required fields saved.
2. Completed user can browse dashboard normally.
3. Routing behavior is stable on refresh and new sessions.
