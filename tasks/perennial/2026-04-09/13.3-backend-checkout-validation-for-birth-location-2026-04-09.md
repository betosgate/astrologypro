Ensure backend checkout API enforces birth-location integrity.

1. Reject missing `birth_location_label`.
2. Reject missing/invalid `birth_lat`, `birth_lng`.

1. Keep location fields in signup payload snapshot.
2. Ensure downstream provisioning receives these keys.

## Acceptance

1. API blocks malformed location data.
2. Valid location data is persisted and carried forward.
