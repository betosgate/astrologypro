## Test Matrix

### A. Signup Phase (Single)

1. Open website and click `GET START`.
2. Verify the first Perennial signup screen shows `Single`, `Couple`, and `Family`.
3. Choose `Single`.
4. Verify the page scrolls to or reveals the Single signup form below.
5. Verify only basic fields shown.
6. Verify `dateOfBirth`, `birthTime`, `birthLocation`, and optional questionnaire are hidden.
7. Complete signup successfully.

### B. First Login Phase

1. Login as newly created single user.
2. Verify redirect to profile-completion form.
3. Verify signup fields are prefilled where available.
4. Verify `dateOfBirth`, `birthTime`, and `birthLocation` are present in the post-login form.
5. Verify required birth fields are empty until user completes them.

### C. City Search Integration

1. Type partial query in birth location input.
2. Verify API search called and list shown.
3. Select one city.
4. Verify `label`, `lat`, `lng`, `tzone` are stored.
5. Verify submit fails if user types only free text and does not select.

### D. Chart Readiness

1. Verify natal generation blocked before required birth fields complete.
2. Complete required fields and verify natal generation allowed.
3. Verify monthly transit depends on natal generated state.

### E. Regression

1. Couple and Family signup behavior unchanged.
2. Existing payment and credential email flow unchanged.
3. No manual password fields introduced.
4. Post-login routing behavior remains unchanged after completion.

## Exit Criteria

1. All A-E checks pass.
2. No redirect loop on login.
3. No data loss between signup and first-login completion.
