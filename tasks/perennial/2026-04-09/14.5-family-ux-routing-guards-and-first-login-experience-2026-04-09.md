## Goal

Guide Family plan users from choose-plan signup to post-login completion without confusion.

## Flow

1. `GET START` -> Perennial choose-plan screen at `/perennial-signup`.
2. User sees `Single`, `Couple`, and `Family` plan cards first.
3. User chooses `Family`, then the page scrolls to or reveals the signup form below.
4. Complete minimal family signup + payment.
5. Login.
6. If completion missing, redirect to family completion form.
7. After completion, allow normal dashboard access.

## Guard Rules

1. Family users missing required fields cannot skip completion.
2. Prevent redirect loops.
3. Completion done => dashboard access.
4. Apply same first-login behavior already used for Single and Couple.
5. Keep the existing post-login experience unchanged after completion.

## Acceptance

1. First login redirects correctly.
2. Completed users are not redirected again.
3. Guard behavior stable across sessions.
