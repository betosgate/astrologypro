## Goal

Define backend prefill and save behavior for Single post-login completion.

## Data Requirements

1. Return signup-captured member profile values for first-login prefill.
2. Store completion fields in canonical member profile storage.
3. Support partial save without overwriting unrelated profile fields.

## Save Rules

1. Save by authenticated `user_id` context only.
2. Do not trust payload `user_id`.
3. Support GET prefill + PATCH save.
4. Return updated profile object and completion state.
5. Persist completion status for routing and readiness gating.
