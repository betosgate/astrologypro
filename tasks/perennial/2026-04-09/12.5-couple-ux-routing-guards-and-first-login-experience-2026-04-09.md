## Goal

Guide couple users from signup to post-login completion without confusion.

## Flow

1. `GET START` -> Perennial choose-plan screen at `/perennial-signup`.
2. User sees `Single`, `Couple`, and `Family` plan cards first.
3. User chooses `Couple`, then the page scrolls to or reveals the signup form below.
4. Complete minimal couple signup + payment.
5. Login.
6. If completion missing, redirect to couple completion form.
7. After completion, allow normal dashboard access.

## Guard Rules

1. Couple users missing required birth fields cannot skip completion.
2. Prevent redirect loops.
3. Completion done => dashboard access.
4. Keep the existing post-login experience unchanged after completion.

## Acceptance

1. First login redirects correctly.
2. Completed users are not redirected again.
3. Guard behavior stable across sessions.
