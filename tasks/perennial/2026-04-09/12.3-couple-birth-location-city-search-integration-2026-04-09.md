## Goal

Reuse same birth-location city search behavior as single-flow for both couple members.

## API Contract

- URL: `https://astrologypro.com/api/admin/astro/city-search`
- Method: `POST`
- Body:
```json
{ "q": "kolk" }
```
- Map selected result to:
1. `birth_location_label`
2. `birth_lat`
3. `birth_lng`
4. `birth_tzone` from `timezone.offset_string`

## UI Rules

1. Debounced search input.
2. Require selecting a result object.
3. No free-text-only submit for birth location.

## Acceptance

1. City selection works for both members.
2. Required birth-location fields are populated from selected result.
