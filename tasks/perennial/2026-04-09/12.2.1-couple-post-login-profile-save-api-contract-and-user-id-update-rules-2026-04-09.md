## Goal

Define save contract for couple post-login completion form.

## Core Rule

1. Save by authenticated `user_id` context only.
2. Do not trust payload `user_id`.

## API Surface

1. Use existing authenticated profile API surface.
2. Support GET prefill + PATCH save.
3. Support incremental partial saves.

## Save Rules

1. GET returns both-member profile object for authenticated owner.
2. PATCH updates the same couple record set for authenticated owner.
3. Return updated object and completion state.
4. Validate required birth fields for both members.

## Acceptance

1. No cross-user overwrite possible.
2. Partial save works.
3. Both-member completion state is persisted.
