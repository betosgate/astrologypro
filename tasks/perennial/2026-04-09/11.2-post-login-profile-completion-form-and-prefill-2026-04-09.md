## Goal

After first login, show a profile completion form for Single-plan users.

## Prefilled Signup Fields (Read/Write)

When the user signs up as Single and logs in for the first time, this form must open with already captured signup values prefilled:

1. `firstName`
2. `lastName`
3. `email`
4. `phone`
5. `gender`
6. `occupation`
7. `address`
8. `city`
9. `state`
10. `zip`

## Required First Completion Fields

1. `date_of_birth`
2. `birth_time`
3. `birth_location_label`
4. `birth_lat`
5. `birth_lng`
6. `birth_tzone`

1. `relationship_status`
2. `mainConcern`
3. `longTermGoals`
4. `personality`
5. `strengths`

## Prefill Behavior

1. Fields captured at signup appear prefilled.
2. New completion fields render empty if missing.
3. User can edit signup fields if needed.

## Data Save Behavior

1. Save partial progress.
2. Mark completion status for gating.
3. Persist in canonical member profile storage.

## Acceptance

1. First login opens completion form automatically.
2. Prefill works for signup-captured values.
3. Completion state persists on refresh/logout/login.
