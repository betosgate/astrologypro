## Goal

Define explicit readiness gates from profile completion to natal chart to monthly transits.

## Readiness Stages

1. `signup_complete`
   - basic single signup done
2. `profile_completion_required`
   - missing required birth fields
3. `natal_ready`
   - required birth fields complete
4. `natal_generated`
   - natal chart exists
5. `transit_ready`
   - natal exists and member active

## Required For Natal

1. `date_of_birth`
2. `birth_time`
3. `birth_location_label`
4. `birth_lat`
5. `birth_lng`
6. `birth_tzone`

## Required For Monthly Transits

1. `natal_chart` exists
2. membership status active

## UI Gating

1. Block monthly transits tab if natal not generated.
2. Show clear CTA to complete profile when not natal-ready.
3. Show clear CTA to generate natal after profile complete.

## Acceptance

1. User cannot reach transit-ready state without natal chart.
2. User cannot generate natal if required birth fields are missing.
3. Dashboard messages match current readiness state.
