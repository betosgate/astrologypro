## Goal

Define backend readiness stages and validation rules for Single natal/transit lifecycle.

## Readiness Stages

1. `signup_complete`
2. `profile_completion_required`
3. `natal_ready`
4. `natal_generated`
5. `transit_ready`

## Required For Natal

1. `date_of_birth`
2. `birth_time`
3. `birth_location_label`
4. `birth_lat`
5. `birth_lng`
6. `birth_tzone`

## Required For Monthly Transits

1. `natal_chart` exists
2. membership status active

## Acceptance

1. User cannot generate natal if required birth fields are missing.
2. User cannot reach transit-ready state without natal chart.
