- Status: Planned
- Priority: P2
- Scope: UI Modal, Next.js Route Handler for Stripe Cancellation

## Objective

Build the legacy "Alert!!" Warning modal that pops up when a user decides to unsubscribe from the Dashboard, and wire it to a custom backend cancellation endpoint.

## Implementation Steps

1.  **UI Component (`<UnsubscribeModal />` & `<MembershipCard />`)**:
    - Open `src/components/community/membership-card.tsx`.
    - Add the **"Subscribed"** button directly to the card (situated horizontally near the "Next Renewal" or replacing the "Manage Billing" slot based on legacy UI).
    - When the user hovers over "Subscribed", it should visually indicate the "Unsubscribe" action.
    - Clicking it opens a dark-themed UI modal with an orange header ("Alert!!") and close mechanism.
    - Match the legacy copy: "Are You Sure You Want To Unsubscribe From The Perennial Mandalism Dashboard?..."
    - Include orange `Unsubscribe` and `Cancel` action buttons.

2.  **API Endpoint (`/api/community/billing/unsubscribe`)**:
    - Create a new POST endpoint replicating the old `unsubscribe-stripe-Perennial` route.
    - Payload to accept: `{ subscription_id: string }`.
    - Secure the route using Supabase auth so that only the authenticated owner can cancel their specific subscription.
    - Use the `stripe.subscriptions.update(id, { cancel_at_period_end: true })` SDK command.

## Done Definition

- The modal perfectly matches the legacy AngularJS styling and copy.
- Pressing "Unsubscribe" safely fires the API route to cancel the Stripe subscription at the period end.
- The UI reflects the "Cancelling" status upon successful API response.
