## Objective

Build a comprehensive "Product List" screen for the Perennial dashboard that allows for administrative management of all products. This screen must match the layout and headers seen in the shared legacy reference image.

## Table Columns (Matches Shared Reference Image 2)

- `#` (Index)
- `Name` (Full name or description)
- `Product Name` (Internal name/slug)
- `Offer Price`
- `Preorder Price`
- `Status` (Active/Inactive)
- `Priority`
- `Visible` (Boolean toggle)
- `Created On` (Datetime)
- `Updated On` (Datetime)
- `Actions`:
    - **Edit** (Icon/Button)
    - **Delete** (Icon/Button)
    - **View Details** (Icon/Button)
    - **Visibility Toggle** (Switch)

## Implementation Steps

1.  **Product Table**:
    - Build `ProductManagementTable.tsx`.
    - Map through the `GET /api/perennial/products` response.
    - Implement column headers with sorting for `Name`, `Price`, `Priority`, and `Date`.

2.  **Product List Page**:
    - Fetch product data on load.
    - Add a "Search" bar for filtering by product name.
    - Add a `Page Size` (10/20/50) and `Page Number` pagination component as seen in the reference.

3.  **Table Inline Actions**:
    - **Edit**: Route to Task 06.2 (`/admin/perennial/products/edit/[id]`).
    - **Delete**: Show a Confirmation Dialog before calling the `DELETE` API.
    - **View**: Open a Modal or Drawer showing full product details.
    - **Visibility Toggle**: Call a PATCH API to instantly update the `is_visible` status.

4.  **Header Actions**:
    - Global "Refresh" icon as seen in the image.
    - Link to "Product Add" form.
