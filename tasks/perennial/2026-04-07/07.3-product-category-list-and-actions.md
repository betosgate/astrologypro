# Task 07.3: Product Category Management - List and Actions - 2026-04-07

- Status: Planned
- Priority: P1

## Objective

Build a comprehensive "Category List" screen for the Perennial dashboard that allows for administrative management of all product categories. This screen will enable efficient listing, editing, deleting and viewing of categories.

## UI Requirements

- **Category Table**:
    - **Columns**: `#` (Index), `Title`, `Description` (Truncated), `Status`, `Priority`, `Created On`, `Actions`.
- **Row Actions**:
    - **Edit**: Icon or button (Navigate to Task 07.2).
    - **Delete**: Show confirmation before removal.
    - **View Details**: Open modal with category overview.
- **Top Actions**:
    - **Search**: Case-insensitive filtering by category title.
    - **Refresh**: Re-fetch current categories from the API.
    - **Add New**: Link to the "Add Category" form.

## Implementation Steps

1.  **Category Management Table Component**:
    - Map through the `GET /api/perennial/product-categories` fetch result.
    - Add status indicators (Active/Inactive) using badges.

2.  **Product Category Dashboard Page**:
    - Organize the page layout with the table and category-specific headers.
    - Ensure pagination works for larger lists.

3.  **Table Inline Actions**:
    - **Delete**: Connect to the `DELETE /api/perennial/product-categories/[id]` endpoint.
    - **View**: Simple presentation of category details.
    - **Edit**: Seamless routing to pre-populated form.
