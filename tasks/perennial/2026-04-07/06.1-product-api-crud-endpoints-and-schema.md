## Objective

Create the back-end infrastructure to support the requested Product Management functionality. This includes defining the database table and building the unified Add/Edit API, Delete API, and Fetch APIs (List and Detail).

## Schema Fields (Matches Shared Reference)

- `id`: UUID (Primary Key)
- `name`: Text (Required)
- `description`: Text (Required)
- `mrp`: Number (Required)
- `offer_price`: Number (Required)
- `preorder_price`: Number (Required)
- `priority`: Integer (Required)
- `category_id`: UUID (Foreign Key to categories)
- `status`: Text (Active/Inactive)
- `main_image_url`: Text (S3/Cloudinary/Storage path)
- `details_image_url`: Text (S3/Cloudinary/Storage path)
- `is_visible`: Boolean (Default: true)
- `created_at`: Timestamptz (Auto)
- `updated_at`: Timestamptz (Auto)

## Implementation Steps

1.  **Database Migration**:
    - Create a SQL migration to add the `perennial_products` table with all fields.
    - Ensure `updated_at` trigger is set.
    - Add a foreign key to any existing category table or create a temporary one.

2.  **Unified Post/Put API (`/api/perennial/products`)**:
    - If `id` is present in the payload, perform `UPDATE`.
    - If `id` is absent, perform `INSERT`.
    - Handle image URL sanitization.

3.  **Delete API (`/api/perennial/products/[id]`)**:
    - Method: `DELETE`.
    - Soft delete or hard delete based on project policy (preferably hard delete if requested).

4.  **Fetch APIs**:
    - **List (`GET /api/perennial/products`)**: Returns all products for the management list, sorted by priority.
    - **Detail (`GET /api/perennial/products/[id]`)**: Returns full details for a single product.

## Done Definition

- Database table exists with correct constraints.
- `POST` / `PUT` endpoint returns 200/201 on success.
- `DELETE` endpoint removes the record.
- `GET` endpoints return valid JSON matching the schema.
- API is protected by admin auth middleware.
